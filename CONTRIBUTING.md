# Contributing

## Workflow
1. Branch from `main`.
2. Keep changes narrowly scoped.
3. Update `README.md` when setup or product scope changes.
4. Run the relevant checks before opening a pull request.

## Expected checks
- `npm install`
- `npm run prisma:generate`
- `npm run build`
- `npm run lint`

## Pull requests
Include:
- problem being solved
- implementation summary
- validation steps
- follow-up work, if any
